# 🤖 بوت إدارة حسابات تيليجرام

بوت تيليجرام احترافي لإضافة وإدارة حسابات تيليجرام متعددة بأمان كامل.

---

## ✨ الميزات

- ➕ إضافة حسابات عبر رقم الهاتف (OTP)
- 🔐 دعم التحقق بخطوتين (2FA)
- 🔒 تشفير الجلسات (AES-256)
- 📋 عرض قائمة الحسابات مع حالتها
- 🔄 إعادة تسجيل الدخول لأي حساب
- 📊 فحص حالة الاتصال لحظيًا
- 🗑️ حذف الحسابات مع تنظيف الجلسات
- 🛡️ معالجة كاملة لأخطاء تيليجرام API
- 💎 نظام اشتراكات تجاري متكامل (باقات، مشتركين، مدفوعات، كوبونات، عروض، إحصائيات)

---

## 🚀 التثبيت

### 1. المتطلبات

- Node.js 18+
- حساب تيليجرام مطور: https://my.telegram.org

### 2. إعداد المشروع

```bash
# نسخ المشروع
git clone <repository-url>
cd telegram-account-bot

# تثبيت المكتبات
npm install

# إعداد متغيرات البيئة
cp .env.example .env
```

### 3. إعداد ملف .env

```env
BOT_TOKEN=        # من @BotFather
API_ID=           # من https://my.telegram.org
API_HASH=         # من https://my.telegram.org
ENCRYPTION_KEY=   # مفتاح عشوائي 32+ حرف
ADMIN_IDS=        # معرّفات تيليجرام (مفصولة بفواصل) لمشرفي نظام الاشتراكات
```

### 4. الحصول على API_ID و API_HASH

1. افتح https://my.telegram.org
2. سجّل دخول برقم هاتفك
3. اضغط "API development tools"
4. أنشئ تطبيقًا جديدًا
5. انسخ `api_id` و `api_hash`

### 5. تشغيل البوت

```bash
# للإنتاج
npm start

# للتطوير
npm run dev
```

## 💎 نظام الاشتراكات

نظام اشتراكات تجاري متكامل ومستقل تمامًا عن باقي أجزاء المشروع، يُضاف كزر جديد "💎 الاشتراكات" في القائمة الرئيسية دون التأثير على أي وظيفة حالية.

### تفعيل صلاحيات الإدارة

أضف معرّفات تيليجرام الخاصة بالمشرفين في `ADMIN_IDS` داخل `.env` (مفصولة بفواصل):

```env
ADMIN_IDS=123456789,987654321
```

- **المشرفون** (معرّفاتهم داخل `ADMIN_IDS`): يرون لوحة تحكم كاملة — إضافة/تعديل/حذف الباقات، إدارة المشتركين، قبول/رفض/استرداد المدفوعات، إنشاء كوبونات وعروض، الإحصائيات، سجل العمليات، والإعدادات.
- **بقية المستخدمين**: يرون واجهة مشترك مبسّطة — تصفّح الباقات، الاشتراك (مع إمكانية استخدام كوبون خصم)، عرض اشتراكهم الحالي، والعروض النشطة.

### آلية الدفع

المدفوعات تُدار يدويًا: عند اختيار المستخدم للاشتراك، يُنشأ طلب "قيد الانتظار" ويصل إشعار فوري لكل المشرفين، ثم يقوم أحدهم بقبول الطلب (فيُفعَّل الاشتراك تلقائيًا) أو رفضه من قسم "💳 إدارة المدفوعات". لا يوجد ربط ببوابة دفع إلكترونية حاليًا.

### ملاحظة حول الحدود (عدد الحسابات / العمليات)

كل باقة تحتفظ بحدّ أقصى لعدد الحسابات والعمليات والمستخدمين، ويظهر هذا الحد للمشرف والمشترك — لكنه **غير مفعَّل تلقائيًا** كقيد فعلي على ميزتَي "📂 الحسابات" و"🔗 الروابط" الحاليتين، حفاظًا على مبدأ عدم المساس بأي وظيفة قائمة. يمكن ربط هذا الحد فعليًا بتلك الميزات في مرحلة لاحقة إذا رغبت بذلك.



- جميع الجلسات مشفرة بـ AES-256
- كلمات المرور تُحذف فورًا من المحادثة
- لا تُخزَّن بيانات الاعتماد في الكود
- المتغيرات الحساسة في ملف `.env` فقط
- التحقق من جميع المدخلات قبل المعالجة
